Gestores de Bases de Datos Relacionales (SQL).

Los sistemas de gestión de bases de datos relacionales (RDBMS) utilizan un modelo basado en tablas para almacenar datos. Estos sistemas utilizan el lenguaje SQL (Structured Query Language) para gestionar y manipular los datos. Algunas características clave son:

- Estructura Tabular: Los datos se organizan en tablas con filas y columnas.
- Relaciones: Permiten establecer relaciones entre diferentes tablas mediante claves primarias y foráneas.
- Transacciones: Soportan transacciones ACID (Atomicidad, Consistencia, Aislamiento, Durabilidad) para garantizar la integridad de los datos.

Ejemplos de RDBMS:
- MySQL: Muy popular y de código abierto, utilizado en aplicaciones web.
- PostgreSQL: Conocido por su robustez y soporte para características avanzadas.
- Oracle Database: Usado en grandes empresas, ofrece características avanzadas de seguridad y rendimiento.
- Microsoft SQL Server: Integrado con el ecosistema de Microsoft, utilizado en muchas aplicaciones empresariales.

Gestores de Bases de Datos No Relacionales (NoSQL).

Los sistemas de gestión de bases de datos no relacionales (NoSQL)
están diseñados para manejar grandes volúmenes de datos que no necesariamente se ajustan a un esquema tabular.
Estos sistemas son más flexibles y escalables. Algunas características clave son:

- Estructura Flexible: Pueden almacenar datos en formatos como documentos, clave-valor, columnas o grafos.
- Escalabilidad Horizontal: Permiten agregar más servidores para manejar un mayor volumen de datos.
- Sin Esquema Fijo: No requieren un esquema predefinido, lo que facilita la adaptación a cambios en los datos.

Tipos de Bases de Datos NoSQL:
1. Bases de Datos de Documentos: Almacenan datos en documentos (generalmente en formato JSON o BSON).

- Ejemplo: MongoDB, CouchDB.
2. Bases de Datos de Clave-Valor: Almacenan datos como pares de clave-valor. - Ejemplo: Redis, Amazon DynamoDB.
3. Bases de Datos de Columna: Almacenan datos en columnas en lugar de filas, lo que permite un acceso más eficiente a grandes volúmenes de datos. Son ideales para análisis de datos y aplicaciones que requieren un alto rendimiento en consultas. - Ejemplo: Apache Cassandra, HBase.
4. Bases de Datos de Grafos: Diseñadas para almacenar y consultar datos que están interconectados, como redes sociales o relaciones entre entidades. Utilizan nodos, aristas y propiedades para representar y almacenar datos. - Ejemplo: Neo4j, Amazon Neptune.
